Veeam Backup & Replication

Upgrade Veeam Backup & Replication 12.3.2.3617 to 13.0.2.229

🎯 Objective

Upgrade an existing Veeam Backup & Replication 12.3 environment to Veeam Backup & Replication 13 while preserving all existing backup jobs, repositories, proxies and configuration.

📋 Environment

Component Value
Operating System Windows Server 2022
Source Version 12.3.2.3617
Target Version 13.0.2.229
Database PostgreSQL
License Enterprise Essentials

⚠️ Prerequisites

  • Verify all backup jobs completed successfully.
  • Verify Configuration Backup is enabled.
  • Verify Configuration Backup files exist on the repository.
  • Verify the license is valid.
  • Create a VMware snapshot.
  • Download the Veeam Backup & Replication 13 ISO.

📦 Verify Configuration Backup

Open:

  • Menu → Configuration Backup

Verify:

  • Configuration Backup is enabled.
  • Last backup completed successfully.
  • The repository is accessible.

Browse the repository and confirm that .BCO files are present.

🔑 Verify License

Open:

  • Menu → License

Verify:

  • License Status = Valid
  • Support contract still active
  • No license warnings

💾 Verify Free Disk Space

Verify the system drive has sufficient free space.

During this upgrade only 6.7 GB remained available on drive C:.

The virtual disk was expanded before starting the upgrade.

The virtual disk was expanded from 80 GB to 150 GB before starting the upgrade.

This step is only required when free space is insufficient.

📸 Create VMware Snapshot

  1. Shut down the Veeam VM.
  2. Create a VMware snapshot.
  3. Power on the VM.

Snapshot name used: Veeam 12 to 13.

💿 Mount Veeam 13 ISO

  1. Download the ISO VeeamBackup & Replication 13 from Veeam customer portail.
  2. Mount the ISO.
  3. Open the mounted DVD drive.
  4. Run Setup.exe.

🚀 Start Upgrade

  1. Click Upgrade Veeam Backup & Replication.
  2. Keep the detected license.
  3. Continue through the wizard.

📦 Install Prerequisites

Veeam 13 requires additional Microsoft components.

The installer automatically installs:

  • Microsoft .NET 8 Hosting Bundle
  • Microsoft Visual C++ Redistributable
  • Microsoft Edge WebView2 Runtime
  • PowerShell 7

Accept installation when prompted.

🔄 Reboot Server

After prerequisite installation Veeam requests a reboot.

  1. Click Reboot.
  2. Wait for the server to restart.
  3. Mount the ISO again if required.
  4. Run Setup.exe.
  5. Resume the upgrade.

⚠️ Compatibility Warnings

The installer may display warnings related to deprecated or discontinued features.

These warnings are informational and do not block the upgrade.

  • Restore Point Retention for new jobs
  • Reverse Incremental Backup for new jobs
  • Single Storage Backup Format
  • Cloud Connect AD Authentication
  • U-AIR Recovery Wizard

Continue if no unsupported operating systems are protected.

⚙️ Upgrade Components

During the upgrade wizard:

  • Enable Update remote components automatically.

This allows Veeam to automatically update backup infrastructure components.

⏳ Perform Upgrade

Click Upgrade and wait for all installation steps to complete.

Expected result:

The wizard should display the message Successfully upgraded.

🔄 Final Reboot

After the upgrade completes, reboot the server when prompted.

🔐 First Login After Upgrade

Launch the Veeam Console.

Select:

Select Sign in as current user.

Complete MFA authentication if required.

🧪 Backup Validation Test

Run an existing backup job manually.

  1. Right-click the backup job.
  2. Select Start.
  3. Do not use Active Full.

Verify the backup job starts and completes successfully.

🔄 Rollback Procedure

Rollback is only required if the upgrade fails or Veeam becomes unusable after the upgrade.

If all post-upgrade validation tests are successful, rollback is not required.

  1. Shut down the VM.
  2. Open VMware Snapshot Manager.
  3. Select the snapshot created before the upgrade.
  4. Click Restore Snapshot.
  5. Power on the VM.
  6. Verify Veeam functionality.
  7. Verify repositories are accessible.
  8. Verify backup jobs are present.

Restore the VMware snapshot created before the upgrade.

🧹 Snapshot Cleanup

Keep the VMware snapshot until at least one backup job has completed successfully after the upgrade.

Once backup validation is successful, delete the VMware snapshot.

  • Do not keep the snapshot longer than necessary.
  • Deleting the snapshot automatically performs disk consolidation.
  • Manual consolidation is only required if VMware displays a "Consolidation Needed" warning.
